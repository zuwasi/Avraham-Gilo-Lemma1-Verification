================================================================================
   7. WHY THE GAP EXISTS (ARCHITECTURAL)
================================================================================

   The gap is not about "intelligence" — it is architectural:

     1. No Symbolic Algebra Engine
        LLMs generate text that looks like math. They predict the next
        token; they do not compute derivatives. When GPT-5 writes
        "d_alpha/dP = ...", it is pattern-matching, not calculating.

     2. No Execution Feedback Loop
        GPT-5 cannot run FindRoot, NIntegrate, or Solve. It cannot plug
        values back in to check if the answer is correct. Each equation
        is generated once with no way to verify it.

     3. Stateless Token Generation
        Each token is predicted from context. There is no internal state
        tracking "I assumed alpha is fixed here, but alpha depends on
        gamma." The model cannot detect when its own assumptions
        contradict its conclusions.

     4. No Self-Consistency Verification
        The critical failure: computing gamma_crit while holding alpha
        fixed, when alpha itself depends on gamma through the FOC.
        Detecting this requires solving an implicit equation — which
        requires a CAS.

   The Ampcode stack closes all four gaps:
     AI reads paper  ->  CandleKeep provides correct syntax  ->
     Mathematica executes  ->  results feed back for self-consistency

================================================================================
   8. BOTTOM LINE

   ChatGPT gave a planning document about how one COULD use Wolfram.
   The Ampcode stack gave a verified proof with an interactive notebook.

   The ChatGPT conversation demonstrates exactly the architectural gap
   that the Ampcode presentation warns about: ChatGPT can TALK about
   using Wolfram, but it cannot EXECUTE, VERIFY, or SELF-CORRECT.
