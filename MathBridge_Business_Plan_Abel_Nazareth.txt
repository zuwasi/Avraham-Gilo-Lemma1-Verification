   1. THE PROBLEM — WOLFRAM IS LOSING USERS TO CHATGPT
================================================================================

   THE THREAT:
     Researchers who need mathematical computation are increasingly
     using ChatGPT instead of Mathematica. Why?

     - ChatGPT: Type English → get answer. No learning curve.
     - Mathematica: Learn Wolfram Language → write code → get answer.

     The result: Wolfram gets $0 from these users.

   THE SCALE:
     - ChatGPT has 300M+ weekly users (OpenAI, 2026)
     - Many use it for math — incorrectly
     - Mathematica has ~303 institutional customers (6sense data)
     - The gap between "people who need math" and "people who use
       Mathematica" is enormous

   WHAT CHATGPT GETS WRONG:
     - No symbolic algebra engine (generates text that looks like math)
     - No execution (cannot verify its own equations)
     - No self-consistency (cannot detect when assumptions contradict
       conclusions)
     - Hallucinates mathematical results

   REAL EXAMPLE — PROF. DAVID GILO, TEL AVIV UNIVERSITY:
     - Law professor studying insurance economics
     - Needed to verify a lemma in his published paper
     - Used ChatGPT: got wrong answer across 5+ attempts
     - ChatGPT computed gamma_crit ≈ 5.9 (incorrect)
     - Correct answer: Lemma 1(i) almost NEVER holds for CRRA
     - ChatGPT found his OLD paper online and used wrong formulas
     - Currently pays Wolfram: $0

================================================================================
   2. THE SOLUTION — MATHBRIDGE
================================================================================

   MathBridge = ChatGPT's Natural Language Interface + Wolfram's CAS

   User experience:
     1. Upload a research paper (PDF)
     2. Type a question in English:
        "Verify Lemma 1 and check if it applies to CRRA utility"
     3. Get a verified answer with:
        - Plain English explanation
        - Interactive plots (sliders, parameters)
        - Downloadable Mathematica .nb notebook
        - Step-by-step proof

   What happens under the hood:
     1. AI reads the paper, extracts equations
     2. AI generates Wolfram Language code (with syntax validation)
     3. Wolfram Engine executes the code
     4. Results feed back to AI for verification
     5. AI explains results in plain English
     6. Self-consistency check catches errors

   DEMO (live, verified):
     https://zuwasi.github.io/Insurance-Demand-Lemma1/presentation/
     Insurance_Demand_Lemma1_Presentation_v2.html
